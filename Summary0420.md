## Raspberry Pi 관련 변동 사항
- 전반적인 구동 흐름 (태블릿 관점)  
    1. 어플에서 인식 요청
    2.  로그인 서버에서 해당 요청 확인 후 파이선 서버에 요청 보냄  
     (로그인 서버 대신 파이선 서버로 직접 보내도됨)
    3. 파이선 서버에서 인식 관련 코드 구동, 결과를 로그인 서버로 전송
    4. 로그인 서버에서 받은 사용자 정보(혹은 약품정보)를 db에 전송, 맞는 값 달라고 요청. db는 등록된 정보를 훑어서 대응하면 응답해줌
    5. db가 전해준 로그인 승인 토큰(jwt)을, 로그인 서버가 어플로 전송
    6. 받은 jwt 기반으로 로그인 진행


- 구동 환경을 바꿈 제의. '카메라'만 담당하게 하고, 안면인식 및 약품인식은 서버 혹은 태블릿에서 구현.
    - 서버에서 구현 : 서버를 크게 두 종류로 묶을 것임. **파이선 서버**(flask 등), node.js를 이용해 **javascript로 짠 서버** 이렇게 둘.  
        - 파이썬 서버(flask 등) : 안면인식 혹은 약품인식 등 Ai기반 딥러닝 관련 부분. 서버의 사양이 핵심. 서버가 인식 후 결과를 로그인 서버로 넘김
        - 로그인 서버(Node.js) : 로그인 관련 서버. 사용자 정보 db에 연결, 파이선 서버에서 받은 사용자 정보와 실제 사용자 정보 대조 후 로그인 승인하는 역할을 함.
    - 어플에서 구현 : **Google ML Kit**를 활용해서, 어플에서 안면인식 관련 딥러닝 코드를 굴림. 이 경우, 필요한 코드의 언어가 바뀌며 어느 정도 수준의 난이도인지 검증이 필요함
- 둘 중 어느쪽이던, 첫 구동 흐름에서 3번 "파이선 서버"가 "어플"로 바뀌는 것 외의 큰 변화는 없음.
- 제(박주현)가 제의하고 싶은 건, 라즈베리의 활용도를 살리긴 해야 하니까 둘 중 한 쪽만 어플, 다른 한 쪽은 서버에 맡기는 것. 실제로 태블릿(전면부에 위치)과 라즈베리 카메라(후면부 혹은 측면부에 위치 가능)의 위치가 다를 수 있고, 관리자 함이 뒤쪽에 위치함을 이용해서 약품인식을 서버(라즈베리 카메라 + 파이선 서버), 안면인식을 어플(태블릿 전면 카메라 + 어플 내부 구동)로 분담하는 건 어떨지?
- 어플에게 학습을 맡기더라도, 구체적으로는 학습 관련 부분을 따로 개발하고 현재 태블릿어플에 붙이는 형태로도 진행 가능. 분업 자체에는 문제가 없으나 팀원분들이 kotlin에 상대적으로 익숙하지 않으심 + 아직 저 ML kit 를 자세히 안 봐서 저도 잘 모름 // 두 가지 이슈로 보류중

## Arduino 관련 사항
- 전반적인 구동 흐름 (태블릿 관점)
    1. 태블릿에서 로그인하거나, 기타 필요한 상황에 아두이노에 "신호"를 보냄
    2. 아두이노는, 아주 직관적으로 "신호"가 들어오면 센서를 움직이게 설정함.  
    이 신호는 서로 조율하면 좋아보임
    3. 아두이노에서 독자적으로 동작하는 부분은, 온도 또는 습도에 맞게 환풍 장치가 동작하는 것. 이 기능을 태블릿에게 맡길 이유가 잘 없음
    4. '필요한 상황'에, 아두이노가 태블릿에게 "신호"를 전달할 필요가 있음.  
    예를 들어, 환풍장치가 가동/정지 하거나(팬의 상태변화), 약품장 센서값에 변화가 생긴 경우(약이 들어있고 없고, 혹은 관리자함의 레벨 센서 등).

- 전송 규격 약속하기 : 
    - 전송 예시 : { 'A':1:1 }
    - 전송 값 정보 : {"장비 이름" : "장비 번호(n번째 칸) : 상태}

        --- 

        #### **장비 이름** :  
        1. 변수 타입 : 알파벳 한 글자로 된 문자(char; "A", "L" 등)
        2. 아두이노 입장에서, 전처리 매크로(#define) 활용해서 직관성 올리면 편함  
        ex) 
        ```C
        #define DEV_ADMIN 'A' // DEV_ADMIN 이라고 적으면 A 로 변신
        printf(DEV_ADMIN);
        ```

        
        #### 장비 번호 :  
        1. 변수 타입 : 정수(int)
        2. 0번부터 시작함에 유의. 처음이 0이고 그 다음부터 1, 기본값은 0
            

        #### 상태 : 사용자함 

        1. 변수 타입 : 정수 
        2. 사용자 함의 경우 이진수 표현으로, 각각의 함의 상태를 1, 또는 0으로 표현함. 맨 왼쪽이 첫번재 칸
            - ex 1) 001010 : 2번째, 4번째 칸 켜짐. 나머지는 꺼짐
            - ex 2) 100110 : 2번째, 3번째, 6번째 칸 켜짐. 나머지는 꺼짐
        3. 값을 수정할 때, XOR 연산으로 비트값을 수정하여 보내는 식
            - ex) 110100 인데, 4번째 비트(0번째~5번째 이므로, 인덱스 3)를 반전해서 보내는 경우. 즉 4번째 칸이 '켜진' 경우  
            ```C
            int currentStatus = 0b110100; // 현재 상태 (예시)  
            int bitIndex = 3;            // 4번째 비트 (인덱스 3)

            int updatedStatus = currentStatus ^ (1 << bitIndex);
            //(1<<3) 은 1을 왼쪽으로 세번 민 수, 즉 0b0001에서 세 칸 민 0b1000을 의미 
            //이러면 updatedStatus 는 0b111100이 됨.
            ```
            이렇게 하거나, 아니면 단순하게  
            
            ```C
            int currentStatus = 0b110100; // 현재 상태 (예시)  
            int bitIndex = 3;            // 4번째 비트 (인덱스 3)

            int updatedStatus1 = currentStatus | (1 << bitIndex); //1으로
            int updatedStatus0 = currentStatus &~(1 << bitIndex); //0으로
            // 각각 4번째 비트를 1,또는 0으로 바꿔치기하는 코드
            ```
            이렇게 해도 댐  
        4. 컴터 입장에서는 001000이랑 8이랑 구분이 안 됨. 디버그 편의성을 위해
            ```C
            print(currentStatus,BIN); //BIN 적어서 2진수 표현으로
            ```
            이런 식으로 하면 8 대신 001000으로 보인다고 함. 아두이노는 앞의 0 날리고 1000 이렇게 보일듯 함  
            좀더 편의성을 붙이고 싶다면 문자열로 바꿔서 해도 되긴 하는데.. 순수 편의성이니 좋으실대로(그냥 20 42 8 이런 십진수 숫자로 보여도 문제는 없음 그냥 아두이노 입장에서의 편의성을 말하는것)

        #### 상태 : 그 외
        - 변수 타입 : 정수 혹은 enum, 온습도는 float
        - enum : 열거형 값. 현재 '관리자 함'에서 쓸 예정
            ```kotlin  
            //kotlin 기준 : 
            enum class TotalStatus {
            EMPTY,   // 0
            LOW,     // 1
            ENOUGH   // 2
            }
            ```
            C언어 버전으로는  
            ```c
            enum TotalStatus {
            EMPTY,   // 0
            LOW,     // 1
            ENOUGH   // 2
            };  

            TotalStatus currentStatus = ENOUGH;
             ```
            - 뭔지 잘 모르겠으면 그냥 정수로 적어도 무방함. 결국 전송하는 값들은 0 1 2 이런 정수일거기 때문에
            - 아니면 앞서 언급한 전처리 매크로(#define) 활용해도 됨.
            - 들어갈 케이스들 소개
                ```c

                #define EMPTY 0
                #define LOW 1
                #define ENOUGH 2
                #define OFF 0
                #define ON 1

                float TempValue = 24.5;
                float HumidValue = 61.9;

                print(TempValue); // 24.5
                int status = OFF; // 0
                printf(status); // 0
                int status = EMPTY; // 0
                println(status); // 얘도 0
                ```
                즉, 0 또는 1 판별만 할 bool이나 0~2 값이 필요한 enum이 공존함. 어차피 0,1,2 이런 정수로 전송될것이기 때문에 '어떤 값을 넣을지'만 코드 작성에서 신경써서 구성하시면 됩니다. 
            - 꺼지면 0, 켜지면 1. 관리자함의 경우는 센서 인식한 값을 읽으셔서 0~2(empty, low, enough을 표현) 중 하나 뱉게 하면 됩니다.
            - 온습도는 float, 다른애들은 enum 혹은 정수인데 문제 안 되나? :  
            어차피 블루투스로 보낼 거라 문자열로 전송됨. 아두이노 자체 코드를 관리할 때만 신경쓰시면 무방할 듯 합니다
            - 아 온습도는 소수 첫째 자리까지만 보내주세요. 크게 중요한 건 아니지만.. 뒤는 잘라주시면 감사
